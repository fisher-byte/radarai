# 4月3日 AI 速报 · 第 171 期

## Summary

Anthropic 将 Computer use 功能正式落地 Windows 平台，标志着 AI 编程智能体向全栈操作系统支持迈出关键一步；与此同时，Google 为 Gemini API 推出 Flex 与 Priority 双服务层级，首次在商用大模型 API 中实现成本弹性与可靠性分级 [1][20]。

## Content

## 🔍 核心洞察  
**Anthropic** 将 **Computer use** 功能正式落地 Windows 平台，标志着 AI 编程智能体向全栈操作系统支持迈出关键一步；与此同时，**Google** 为 **Gemini API** 推出 **Flex** 与 **Priority** 双服务层级，首次在商用大模型 API 中实现成本弹性与可靠性分级 [1][20]。

## 🚀 重点动态  
- **Claude 的 Computer use 功能现已支持 Windows** [1]：Anthropic 面向 Windows 用户开放 Claude Cowork 与 Claude Code Desktop 的自动化任务能力。  
- **Google 为 Gemini API 引入 Flex 和 Priority 服务层级** [20]：新增“flex”降本模式与“priority”高可靠模式，支持差异化生产部署。  
- **Cursor 3 发布：整合模型、运行时与界面** [4]：Cursor 宣布转型为端到端智能体编程平台，统一提供模型调度、执行环境与开发者界面。  
- **Pika 为 AI Agent 推出实时视频通话功能** [5]：测试版支持 AI Agent 带持久记忆与个性接入 Google Meet 实时会议。  
- **Codex 更新定价策略，提升团队使用便捷性** [2]：转向按用量计费，取消前期承诺，显著降低企业级采用门槛。  
- **KernelEvolve：Meta 的排序工程师智能体如何优化 AI 基础设施** [17]：该智能体系统自动生成并优化异构加速器内核，在生产负载中实现显著性能提升。  
- **LMArena 发布完整的 Arena 排行榜数据集** [19]：开源三年历史评测数据，推动大模型评估方法论的透明化与可复现研究。  
- **OpenAI 收购 TBPN 的战略分析** [3]：收购逻辑聚焦于 TBPN 直播/未经剪辑的内容形态，以对抗 AI 生成内容的同质化风险。
